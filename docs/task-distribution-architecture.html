simple-a2a-registry-v2 · 任务分发架构

分层架构：事件驱动总线 + 声明式调度引擎 + 三层 Worker 分发

Phase 1 — 事件驱动化

  • • 引入 asyncio.Queue 消除 5s 轮询延迟
  • • 写操作后发布事件 → 立即触发 dispatch
  • • 保留 30s 轮询兜底防丢事件
  • • 影响范围：store.py + dispatcher.py

Phase 2 — 状态持久化

  • • 新增 AgentFlowState 表（MySQL/SQLite）
  • • FlowController 改为 Read-Through Cache
  • • 熔断器状态持久化，重启不丢失
  • • 影响范围：store.py + flow_controller.py

Phase 3 — WorkerExecutor 抽象

  • • 定义 WorkerExecutor 抽象基类
  • • 多实现：Local / K8s / Docker / Callback
  • • Dispatcher 通过配置选择 Executor
  • • 影响范围：executor.py + dispatcher.py
